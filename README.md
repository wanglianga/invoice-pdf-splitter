# 发票 PDF 拆分命名工具

一款基于 Tauri 的本地发票 PDF 拆分命名桌面应用，专为财务人员、报销员工和审计人员设计，用于整理混合发票文件。

## 原始需求

> 扩展多类型页面识别：一个 PDF 里混有增值税发票、行程单、报销封面、空白页和附件时，工具要区分可拆分页、需人工判断页、合并附件页和无效页，用户确认后再批量输出。
> 扩展命名模板预览：财务设置日期、金额、销售方、发票号、报销人或项目编号组合时，界面实时展示生成文件名，提示非法字符、路径过长、重复名称和缺失字段。

## 功能特性

- 📁 **拖放支持**：支持拖入多页 PDF 文件或文件夹，自动扫描目录中的 PDF
- 🔍 **智能识别**：自动识别发票号码、购买方、销售方、金额、日期、票种等信息
- 🏷️ **多类型页面分类**：自动区分增值税发票、行程单、报销封面、附件、空白页，并标记操作类型（可拆分/需人工判断/合并附件/无效页）
- ✂️ **PDF 拆分**：将多页 PDF 按页拆分为独立的单页 PDF 文件
- 📝 **自定义命名**：支持自定义命名模板，可使用多种变量
- 👁️ **命名预览**：实时预览生成文件名，提示非法字符、路径过长、重复名称和缺失字段
- 📊 **进度显示**：实时显示处理进度、当前文件、页码和状态
- ⚙️ **灵活配置**：重复文件处理方式（覆盖/跳过/自动重命名）
- 🔧 **人工校正**：支持对识别结果进行人工校正
- 📋 **处理日志**：完整记录处理过程和结果
- 🔒 **本地处理**：所有文件均在本地处理，数据不上传，保护隐私

## 技术栈

- **前端**：React 18 + TypeScript + Vite
- **后端**：Rust + Tauri 2.0
- **PDF 处理**：lopdf
- **文本识别**：正则表达式匹配

## 启动方式

### 前置要求

- Node.js >= 18
- Rust >= 1.70
- 系统依赖（Windows）：WebView2（Windows 10/11 已预装）

### 启动步骤

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动开发模式

```bash
npm run tauri dev
```

访问地址：应用窗口自动弹出（Tauri 桌面应用）

首次启动会编译 Rust 依赖，可能需要较长时间。

#### 3. 构建生产版本

```bash
npm run tauri build
```

构建完成后，安装包位于 `src-tauri/target/release/bundle/` 目录。

### 使用说明

1. **添加文件**：将 PDF 文件或文件夹拖入左侧区域，或点击选择文件/文件夹
2. **识别页面类型**：点击"识别页面类型"按钮，自动分类各页面的类型和操作
3. **审核分类结果**：在"页面分类"标签页查看分类结果，可修改页面操作类型和发票信息
4. **设置输出**：选择输出目录，配置命名模板，查看文件名预览和警告
5. **确认批量输出**：审核完成后点击"确认并批量输出"按钮
6. **查看结果**：在右侧面板查看处理结果、发票信息和处理日志

### 命名模板变量

| 变量 | 说明 |
|------|------|
| `{发票号码}` | 发票号码 |
| `{购买方}` | 购买方名称 |
| `{销售方}` | 销售方名称 |
| `{金额}` | 发票金额 |
| `{日期}` | 开票日期 |
| `{票种}` | 发票类型 |
| `{原页码}` | 原 PDF 中的页码 |
| `{报销人}` | 报销人/申请人/经办人 |
| `{项目编号}` | 项目编号/合同编号 |

默认模板：`{日期}_{销售方}_{发票号码}_{金额}`

### 页面类型分类

| 类型 | 默认操作 | 说明 |
|------|----------|------|
| 增值税发票 | 可拆分 | 识别为增值税专用/普通发票的页面 |
| 行程单 | 可拆分 | 机票行程单、火车票、出租车票等 |
| 报销封面 | 需人工判断 | 报销单、费用报销单等封面页 |
| 附件 | 合并附件 | 银行回单、合同、明细表等 |
| 空白页 | 跳过（无效） | 内容极少或完全空白的页面 |
| 未知 | 需人工判断 | 无法确定类型的有内容页面 |

### 命名验证警告

| 警告类型 | 图标 | 说明 |
|----------|------|------|
| missing_field | ❓ | 模板包含某变量但该字段未识别 |
| illegal_char | ⚠️ | 文件名包含非法字符 |
| path_too_long | 📏 | 文件名超过 200 字符 |
| duplicate_name | 🔄 | 多个页面生成相同文件名 |
| placeholder_in_name | 📋 | 文件名中包含"未识别"等占位文本 |

## Docker 启动方式

> 注意：Tauri 是桌面应用程序，Docker 容器中无法直接运行 GUI。Docker 配置主要用于构建应用和 CI/CD 环境。

### 前置要求

- Docker >= 20.10
- Docker Compose >= 2.0

### 构建应用

```bash
docker compose up --build
```

或后台运行：

```bash
docker compose up --build -d
```

停止构建容器：

```bash
docker compose down
```

Docker 容器将执行以下操作：
- 安装所有依赖
- 构建前端静态资源
- 编译 Rust 后端
- 输出构建产物

### 构建产物

构建完成后，可在容器内的 `/app/src-tauri/target/release/` 目录找到编译产物。

如需从容器中复制产物，可使用：

```bash
docker cp invoice-splitter:/app/src-tauri/target/release/ ./build-output/
```

## 注意事项

1. **识别准确度**：发票信息和页面类型识别基于正则表达式匹配，对于格式特殊的发票可能识别不准确，建议使用页面分类功能审核后再批量输出
2. **PDF 兼容性**：部分加密或格式特殊的 PDF 可能无法正常读取
3. **性能**：处理大量文件时建议分批处理
4. **数据安全**：所有处理均在本地完成，不会上传任何数据

## 常见问题

**Q: 识别结果不准确怎么办？**

A: 可以使用"识别页面类型"功能，审核分类结果后修改页面操作类型和发票信息，确认后再批量输出。也可以开启"人工校正"功能手动修正。

**Q: 支持哪些页面类型？**

A: 支持增值税发票、行程单、报销封面、附件、空白页五种类型自动分类，未识别的类型标记为"未知"需人工判断。

**Q: 命名模板中显示警告怎么处理？**

A: 警告提示了非法字符、路径过长、重复名称和缺失字段等问题。建议根据警告调整命名模板或补充缺失字段，避免输出时文件名异常。

**Q: 处理后的文件保存在哪里？**

A: 保存在您设置的输出目录中，默认为空，需要手动选择。
