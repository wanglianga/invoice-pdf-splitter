# 发票 PDF 拆分命名工具

一款基于 Tauri 的本地发票 PDF 拆分命名桌面应用，专为财务人员、报销员工和审计人员设计，用于整理混合发票文件。

## 原始需求

> 请实现本地发票 PDF 拆分命名工具，Tauri 应用给财务、报销员工和审计人员整理一批混合发票文件。工具支持拖入多页 PDF 或文件夹，按页预览，识别发票号码、购买方、销售方、金额、日期、票种和报销附件，把每张发票拆成独立文件并按规则命名。用户可以设置命名模板、输出目录、重复文件处理方式、人工校正和处理日志；所有 PDF 留在本机，拆分过程显示进度、失败原因、原页码和改名结果。

## 功能特性

- 📁 **拖放支持**：支持拖入多页 PDF 文件或文件夹，自动扫描目录中的 PDF
- 🔍 **智能识别**：自动识别发票号码、购买方、销售方、金额、日期、票种等信息
- ✂️ **PDF 拆分**：将多页 PDF 按页拆分为独立的单页 PDF 文件
- 📝 **自定义命名**：支持自定义命名模板，可使用多种变量
- 📊 **进度显示**：实时显示处理进度、当前文件、页码和状态
- ⚙️ **灵活配置**：重复文件处理方式（覆盖/跳过/自动重命名）
- 🔧 **人工校正**：支持对识别结果进行人工校正
- 📋 **处理日志**：完整记录处理过程和结果
- 🔒 **本地处理**：所有文件均在本地处理，数据不上传，保护隐私

## 技术栈

- **前端**：React 18 + TypeScript + Vite
- **后端**：Rust + Tauri 2.0
- **PDF 处理**：lopdf
- **文本识别**：正则表达式匹配

## 启动方式

### 前置要求

- Node.js >= 18
- Rust >= 1.70
- 系统依赖（Windows）：WebView2（Windows 10/11 已预装）

### 启动步骤

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动开发模式

```bash
npm run tauri dev
```

首次启动会编译 Rust 依赖，可能需要较长时间。

#### 3. 构建生产版本

```bash
npm run tauri build
```

构建完成后，安装包位于 `src-tauri/target/release/bundle/` 目录。

### 使用说明

1. **添加文件**：将 PDF 文件或文件夹拖入左侧区域，或点击选择文件/文件夹
2. **设置输出**：选择输出目录，配置命名模板和重复文件处理方式
3. **开始处理**：点击"开始处理"按钮，等待处理完成
4. **查看结果**：在右侧面板查看处理结果、发票信息和处理日志
5. **人工校正**：如有需要，可对识别结果进行人工校正

### 命名模板变量

| 变量 | 说明 |
|------|------|
| `{发票号码}` | 发票号码 |
| `{购买方}` | 购买方名称 |
| `{销售方}` | 销售方名称 |
| `{金额}` | 发票金额 |
| `{日期}` | 开票日期 |
| `{票种}` | 发票类型 |
| `{原页码}` | 原 PDF 中的页码 |

默认模板：`{日期}_{销售方}_{发票号码}_{金额}`

## Docker 启动方式

> 注意：Tauri 是桌面应用程序，Docker 容器中无法直接运行 GUI。Docker 配置主要用于构建应用和 CI/CD 环境。

### 前置要求

- Docker >= 20.10
- Docker Compose >= 2.0

### 构建应用

```bash
docker compose build
```

### 运行构建

```bash
docker compose up
```

或后台运行：

```bash
docker compose up -d
```

停止构建容器：

```bash
docker compose down
```

Docker 容器将执行以下操作：
- 安装所有依赖
- 构建前端静态资源
- 编译 Rust 后端
- 输出构建产物

### 构建产物

构建完成后，可在容器内的 `/app/src-tauri/target/release/` 目录找到编译产物。

如需从容器中复制产物，可使用：

```bash
docker cp invoice-splitter:/app/src-tauri/target/release/ ./build-output/
```

## 注意事项

1. **识别准确度**：发票信息识别基于正则表达式匹配，对于格式特殊的发票可能识别不准确，建议开启人工校正模式
2. **PDF 兼容性**：部分加密或格式特殊的 PDF 可能无法正常读取
3. **性能**：处理大量文件时建议分批处理
4. **数据安全**：所有处理均在本地完成，不会上传任何数据

## 常见问题

**Q: 识别结果不准确怎么办？**

A: 可以开启"人工校正"功能，在处理完成后手动修正识别结果。也可以反馈问题帮助我们改进识别算法。

**Q: 支持哪些发票类型？**

A: 支持增值税专用发票、增值税普通发票、电子发票等常见发票类型，其他类型发票可能需要人工校正。

**Q: 处理后的文件保存在哪里？**

A: 保存在您设置的输出目录中，默认为空，需要手动选择。
